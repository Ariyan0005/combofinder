---
name: Importing an existing GitHub repo into a Replit artifact-based repl
description: How to replace a stub/scaffold artifact with the real contents of a pre-existing multi-service repo, when the repl already has artifact.toml-registered services.
---

When a user's real project lives in an external GitHub repo and the current
Replit repl only has stub/scaffold artifacts (created by earlier
onboarding), importing the real repo means replacing each artifact's
directory contents wholesale, then re-registering the artifact so its
`artifact.toml`/workflow stay consistent.

**Recipe that worked:** for each artifact directory, move the existing
stub aside, copy in the real directory from the cloned repo, then use the
artifacts tooling to re-create/restore the artifact registration (rather
than hand-editing `artifact.toml`) so the workflow and preview routing
regenerate correctly. Clone the source repo read-only to a scratch path
(e.g. `/tmp/<name>_check`) first so you can diff/copy without touching git
state in the workspace.

**Why:** artifact.toml and its workflow are managed/generated by the
artifact tooling; wholesale directory replacement without going through
that tooling risks leaving the workflow pointing at stale config.

**How to apply:** after import, restart every affected workflow and
screenshot each artifact's running preview to confirm it shows real
product content (not leftover scaffold placeholders) before doing any
feature work on top of it.
